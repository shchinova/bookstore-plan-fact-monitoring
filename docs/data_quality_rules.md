# Правила проверки качества данных

> Документ описывает правила валидации и очистки данных, импортируемых из 1С. Правила применяются при первичной загрузке (`load_history.py`) и при каждом ежедневном обновлении (`daily_update.py`). При изменении правил сначала обновляется этот документ, затем — код `scripts/validate.py`.

## Справочник допустимых значений

```
GENRES   = ['Fiction', 'Fantasy', 'Mystery', 'Romance', 'Science',
            'Biography', 'Business', 'Children', 'History', 'Poetry', 'Subscription']

FORMATS  = ['eBook', 'Paperback', 'Hardcover', 'Audiobook', 'Subscription']

CHANNELS = ['Web', 'Mobile App', 'API Partner']

LOW_STOCK_THRESHOLD = 15   # порог для флага is_low_stock
```

## Общие правила

|Код|Правило|Действие|
|---|---|---|
|G-01|Разделитель столбцов — запятая (`,`). Строки с иным разделителем (например, `;`) считаются некорректными.|Отклонить строку, записать в отчёт|
|G-02|Кодировка файла — UTF-8. Строки с нечитаемыми символами (артефакты CP1251) считаются повреждёнными.|Попытаться восстановить; если не удалось — отклонить строку, записать в отчёт|
|G-03|Пустая строка (`' '`, `''`) в текстовом поле приравнивается к `NULL`.|Заменить на `NULL`, записать в отчёт|
|G-04|Лишние пробелы в начале и конце строкового значения.|Обрезать (`strip()`), записать количество исправлений в отчёт|
|G-05|Полностью пустые строки (все поля `NULL` или пустые).|Удалить, записать в отчёт|

## `dim_date` — Календарь

Поля: `date`, `year`, `month`, `quarter`, `is_weekend`, `is_holiday_ru`.
Правила:

|Код|Поле|Правило|Действие|
|---|---|---|---|
|D-01|`date`|Формат даты — `YYYY-MM-DD`. Любое отклонение считается ошибкой.|Попытаться привести к стандарту; если не удалось — отклонить строку|
|D-02|`date`|Значение должно быть валидной календарной датой.|Отклонить строку|
|D-03|`date`|Дубликаты по полю `date` недопустимы.|Удалить дубликат, оставить первую запись, записать в отчёт|
|D-04|`year`, `month`, `quarter`|Должны соответствовать значению `date`. Например, для `2024-03-15`: `year=2024`, `month=3`, `quarter=1`.|Пересчитать из `date`, записать расхождения в отчёт|
|D-05|`is_weekend`|Булево (0 или 1). Должно соответствовать `date`: 1 если weekday ≥ 5 (сб/вс), иначе 0.|Пересчитать из `date`, записать расхождения в отчёт|
|D-06|`is_holiday_ru`|Булево (0 или 1).|Отклонить строку при недопустимом значении|

## `dim_product` — Справочник продуктов

Поля: `product_id`, `isbn`, `title`, `author`, `publisher`, `published_date`, `language`, `page_count`, `genre`, `format`, `price_rub`, `cost_rub`, `avg_rating`, `review_count`, `stock_initial`, `is_physical`.
Правила:

|Код|Поле|Правило|Действие|
|---|---|---|---|
|P-01|`product_id`|Уникальный идентификатор, `NOT NULL`, целое число > 0. Дубликаты по `product_id` недопустимы.|Отклонить дублирующую запись, записать в отчёт|
|P-02|`title`|`NOT NULL`, непустая строка.|Отклонить строку|
|P-03|`author`|Ожидается непустая строка. `NULL` допустим только для подписок (`format = 'Subscription'`).|Записать в отчёт как предупреждение|
|P-04|`publisher`|Ожидается непустая строка. `NULL` допустим только для подписок.|Записать в отчёт как предупреждение|
|P-05|`format`|Допустимые значения: `eBook`, `Paperback`, `Hardcover`, `Audiobook`, `Subscription`.|Отклонить строку|
|P-06|`genre`|Допустимые значения: см. справочник `GENRES`.|Отклонить строку|
|P-07|`isbn`|Для книг: валидный ISBN-13 (13 цифр, корректная контрольная сумма). Для подписок (`format = 'Subscription'`): `NULL`.|Записать в отчёт как предупреждение (не блокирует загрузку)|
|P-08|`price_rub`|`NOT NULL`, числовое, > 0.|Отклонить строку|
|P-09|`cost_rub`|`NOT NULL`, числовое, > 0.|Отклонить строку|
|P-10|`cost_rub`|Себестоимость не должна превышать цену (`cost_rub ≤ price_rub`).|Записать в отчёт как предупреждение|
|P-11|`page_count`|Для `Paperback` и `Hardcover`: ожидается целое число > 0, `NULL` — предупреждение. Для остальных форматов: ожидается `NULL`.|Записать в отчёт как предупреждение|
|P-12|`stock_initial`|Для `eBook`, `Audiobook`, `Subscription`: должен быть `NULL`. Ненулевое значение — ошибка логики.|Обнулить (`NULL`), записать в отчёт|
|P-13|`avg_rating`|Числовое, диапазон 0.0–5.0 включительно.|Отклонить строку|
|P-14|`review_count`|Целое, ≥ 0.|Отклонить строку|
|P-15|`is_physical`|Булево (0 или 1). Должно соответствовать `format`: 1 для `Paperback`/`Hardcover`, 0 для остальных.|Пересчитать из `format`, записать расхождения в отчёт|
|P-16|—|Дубликат записи: совпадение по всем полям кроме `product_id`. Признак задвоения в 1С.|Загрузить только первую запись, записать в отчёт как предупреждение|

## `fact_plan` — Планы

Поля: `plan_id`, `genre`, `format`, `year`, `month`, `plan_qty`, `plan_amount`, `plan_margin_target`.
Правила:

|Код|Поле|Правило|Действие|
|---|---|---|---|
|PL-01|`plan_id`|Уникальный идентификатор, `NOT NULL`, целое число > 0.|Отклонить дубликат|
|PL-02|`genre`|Допустимые значения: см. справочник `GENRES` (без `Subscription` — подписки не планируются по жанру).|Отклонить строку|
|PL-03|`format`|Допустимые значения: `eBook`, `Paperback`, `Hardcover`, `Audiobook` (без `Subscription`).|Отклонить строку|
|PL-04|`year`|Целое число, в диапазоне периода проекта: 2024–2026.|Отклонить строку|
|PL-05|`month`|Целое число, 1–12.|Отклонить строку|
|PL-06|`plan_qty`|Числовое, ≥ 0. Отрицательные значения недопустимы.|Отклонить строку|
|PL-07|`plan_amount`|Числовое, ≥ 0. Отрицательные значения недопустимы.|Отклонить строку|
|PL-08|`plan_qty`, `plan_amount`|Если `plan_qty = 0`, то `plan_amount` должен быть 0, и наоборот.|Записать в отчёт как предупреждение|
|PL-09|`plan_margin_target`|Числовое, диапазон 0.0–1.0 включительно.|Отклонить строку|
|PL-10|`(genre, format, year, month)`|Комбинация должна быть уникальной. Дубликат плана на один период недопустим.|Удалить дубликат, оставить первую запись, записать в отчёт|

## `fact_sales` — Продажи

Поля: `sales_id`, `order_id`, `product_id`, `date`, `sales_qty`, `return_qty`, `unit_price`, `sales_amount`, `return_amount`, `discount_percent`, `is_promo`, `promo_code`, `channel`, `lost_sales_qty`
Правила:

|Код|Поле|Правило|Действие|
|---|---|---|---|
|S-01|`sales_id`|Уникальный идентификатор, `NOT NULL`. Дубликаты недопустимы.|Удалить дубликат, записать в отчёт|
|S-02|`order_id`|`NOT NULL`.|Записать строки с `NULL` в отчёт как предупреждение|
|S-03|`(order_id, product_id)`|Комбинация должна быть уникальной в рамках одного заказа.|Удалить дубликат, записать в отчёт|
|S-04|`product_id`|Должен существовать в `dim_product`.|Отклонить строку|
|S-05|`date`|Формат `YYYY-MM-DD`, валидная дата.|Попытаться привести; если не удалось — отклонить|
|S-06|`date`|Дата в пределах периода проекта: `2024-01-01` — текущая дата.|Отклонить строку|
|S-07|`date`|Проверка полноты: если за какой-либо день в периоде нет ни одной строки продаж — потерянный день.|Записать в отчёт как предупреждение|
|S-08|`sales_qty`|Целое, > 0.|Отклонить строку|
|S-09|`unit_price`|Числовое, > 0.|Отклонить строку|
|S-10|`sales_amount`|Ожидаемое значение: `sales_qty × unit_price`. Расхождение более 1% — ошибка. Примечание: `unit_price` уже рассчитан с учётом скидки (`price_rub × (1 − discount_percent / 100)`).|Записать в отчёт как предупреждение|
|S-11|`return_qty`|Целое, ≥ 0. Для цифровых товаров (`eBook`, `Audiobook`, `Subscription`): ожидается 0.|Записать ненулевые значения для цифровых в отчёт как предупреждение|
|S-12|`return_qty`|Не должен превышать `sales_qty`.|Записать в отчёт как предупреждение|
|S-13|`return_amount`|Числовое, ≥ 0. Ожидаемое значение: `return_qty × unit_price`. Если `return_qty = 0`, то `return_amount = 0`.|Записать расхождения в отчёт как предупреждение|
|S-14|`discount_percent`|Числовое, диапазон 0–100 включительно.|Отклонить строку|
|S-15|`is_promo`|Булево (0 или 1). Если `is_promo = 1`: `discount_percent > 0` и `promo_code NOT NULL`.|Записать нарушения в отчёт как предупреждение|
|S-16|`is_promo`|Если `is_promo = 0`: `discount_percent = 0`.|Записать нарушения в отчёт как предупреждение|
|S-17|`channel`|Допустимые значения: `Web`, `Mobile App`, `API Partner`.|Отклонить строку|
|S-18|`lost_sales_qty`|Целое, ≥ 0. Для цифровых товаров: ожидается 0.|Записать в отчёт как предупреждение|

## `fact_inventory` — Остатки

Поля: `inventory_id`, `product_id`, `date`, `opening_stock`, `sold_qty`, `replenishment_qty`, `closing_stock`, `is_low_stock`
Правила:

|Код|Поле|Правило|Действие|
|---|---|---|---|
|I-01|`product_id`|Должен существовать в `dim_product`.|Отклонить строку|
|I-02|`product_id`|Только физические книги: `format IN ('Paperback', 'Hardcover')`.|Отклонить строку|
|I-03|`date`|Формат `YYYY-MM-DD`, валидная дата.|Попытаться привести; если не удалось — отклонить|
|I-04|`date`|Дата в пределах периода проекта: `2024-01-01` — текущая дата.|Отклонить строку|
|I-05|`(product_id, date)`|Комбинация должна быть уникальной.|Удалить дубликат, записать в отчёт|
|I-06|`opening_stock`|Целое, ≥ 0.|Отклонить строку|
|I-07|`sold_qty`|Целое, ≥ 0.|Отклонить строку|
|I-08|`replenishment_qty`|Целое, ≥ 0.|Отклонить строку|
|I-09|`closing_stock`|Целое, ≥ 0. Отрицательные остатки недопустимы.|Отклонить строку|
|I-10|`closing_stock`|Логический баланс: `closing_stock = opening_stock − sold_qty + replenishment_qty`.|Записать расхождения в отчёт как предупреждение|
|I-11|`is_low_stock`|Булево (0 или 1). Должно соответствовать условию: `is_low_stock = 1` если `closing_stock < 15`, иначе `0`. Порог фиксирован (`LOW_STOCK_THRESHOLD = 15`).|Пересчитать, записать расхождения в отчёт|

## Формат отчёта о качестве данных

Отчёт сохраняется в `reports/` в формате Markdown.  
Имя файла: `quality_report_YYYY-MM-DD_HH-MM.md`  
Каждый запуск создаёт новый файл.

### Структура отчёта

```markdown

Дата запуска: YYYY-MM-DD HH:MM
Скрипт: load_history.py / daily_update.py

## Сводка

|----------------|----------------|-----------|------------|----------------|-----------|
| dim_date       | ...            | ...       | ...        | ...            | ...       |
| dim_product    | ...            | ...       | ...        | ...            | ...       |
| fact_plan      | ...            | ...       | ...        | ...            | ...       |
| fact_sales     | ...            | ...       | ...        | ...            | ...       |
| fact_inventory | ...            | ...       | ...        | ...            | ...       |

## Детали по таблицам
```

## Приоритеты действий

|Приоритет|Действие|Когда применяется|
|---|---|---|
|🔴 Отклонить строку|Строка не загружается в БД|Критическая ошибка: нарушение типа, недопустимое значение, висящий FK|
|🟡 Исправить и загрузить|Значение исправляется автоматически|Восстановимый формат даты, пустая строка → NULL, пересчёт производного поля|
|🔵 Предупреждение|Строка загружается, аномалия фиксируется|Логическое несоответствие, отсутствие необязательного поля|
